# 06 — Gap Analysis (cross-cutting, severity-rated, acceptance-criteria)

**Status:** Draft · 2026-06-18 · conforms to `_shared-strategic-brief.md` (Model 2,
staged property OS). This document grounds every gap in real files/routes — it does
**not** invent missing work. Where a control already exists, that is stated and the
gap is reduced to "verify/tighten" rather than "build".

**Headline:** Propvora is materially *further along* than a typical pre-beta SaaS.
The schema-alignment gate is green (`node scripts/audit-queries.mjs` → "0 misaligned
column refs"), 183 unit tests pass across 13 files (`vitest run`), and the hard
infra (Stripe webhook idempotency, R2 magic-byte upload validation, GDPR erasure
worker, DSAR, rate-limit, maintenance mode, observability scaffold) is already
present. The dominant risks are therefore **operational wiring** (no observability
sink registered, AI hard-cap deferred, silent email no-op), **strategic surface
discipline** (settings/calendar sprawl, marketplace flag-leak), and **release
evidence** (DR runbooks are templated stubs, no browser/e2e gate in CI), **not**
greenfield feature build.

Severity key: **Critical** = blocks paid launch / data-loss / money-loss / legal
exposure · **High** = blocks beta or erodes trust quickly · **Med** = should fix in
launch window · **Low** = post-launch polish.

Timing key: **B** = fix before beta · **P** = fix before paid launch · **L** =
post-launch.
